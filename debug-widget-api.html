🔍 ProofLayer Widget API Debugger

This tool helps diagnose widget loading issues

Step 1: Test Your API Endpoint

Format: {YOUR_APP_URL}/api/widget/{WORKSPACE_ID}
Example: http://localhost:3000/api/widget/abc-123-def-456

Step 2: Test Widget.js Load

Step 3: Pre-Flight Checklist

  • Is your dev server running? npm run dev
  • Have you created a workspace in the dashboard?
  • Have you created at least one form?
  • Have you submitted at least one testimonial?
  • Have you approved at least one testimonial? (Status must be APPROVED, not PENDING)
  • Is your workspace ID correct in the embed code?
  • Are you testing on the same domain (localhost) or have CORS enabled?

Step 4: Common Issues

❌ "Failed to load testimonials"
Causes:
  • API endpoint returns error (workspace not found, database error)
  • Network error or CORS blocking the request
  • Workspace ID is incorrect
⚠️ "No testimonials yet"
Causes:
  • No testimonials have been submitted
  • Testimonials exist but none are APPROVED (still PENDING or REJECTED)
  • API is working but returning empty array
❌ "Container #prooflayer-widget not found"
Causes:
  • Missing <div id="prooflayer-widget"></div>
  • Script loads before the div (put div before script)
❌ "Missing data-workspace attribute"
Causes:
  • Script tag missing data-workspace="..." attribute
  • Workspace ID is empty string

Step 5: Database Check Query

Run this in your database to check for approved testimonials:

SELECT w.id as workspace_id, w.name as workspace_name, COUNT(s.id) as approved_count
FROM workspaces w
LEFT JOIN forms f ON f.workspace_id = w.id
LEFT JOIN submissions s ON s.form_id = f.id AND s.status = 'APPROVED'
GROUP BY w.id, w.name
ORDER BY w.created_at DESC;
